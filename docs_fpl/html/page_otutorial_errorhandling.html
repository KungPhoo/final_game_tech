Get latest error

In case something goes wrong you can always call fpl::GetPlatformError() - at any time.
This either returns a empty string indicating everything is fine or a constant string with a valid error message.

Example:

const char *errStr = fpl::GetPlatformError();
// Do something with the error string

Was there a error?

If you just want to check if there was an error, you can call fpl::GetPlatformErrorCount() to use the number of errors as a condition.

Example:

if (fpl::GetPlatformErrorCount()) {
// Print out the error message
}

Clearing the errors

Errors will never be cleared by FPL! You have to do this yourself using fpl::ClearPlatformErrors() .

Example:

fpl::ClearPlatformErrors();

Note

Keep in mind that any platform error is reported in the output of the error console automatically!
If you dont want to do that, then simply define the preprocessor directive for ignoring this behavior - before including this library with implementation enabled:

#define FPL_NO_ERROR_IN_CONSOLE
#define FPL_IMPLEMENTATION
#include <final_platform_layer.hpp>
