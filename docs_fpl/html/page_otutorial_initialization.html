Include the Library

In one of your C++ translation units include this:

#define FPL_IMPLEMENTATION
#include "final_platform_layer.hpp"

You can then include this file in any other C++ source or header file as you would with any other header file.

Entry Point

Simply provide the typical main entry point (char, C++ style):

int main(int argc, char *args[]) {
// code goes here
}

Initialization

To initialize FPL you need to call fpl::InitPlatform() (inside the main Entry Point) and provide the desired fpl::InitFlags :

int main(int argc, char *args[]) {
// With defaults (Window, Video, Audio)
fpl::InitPlatform(fpl::InitFlags::All);
// Only audio
fpl::InitPlatform(fpl::InitFlags::Audio);
// Only window and audio
fpl::InitPlatform(fpl::InitFlags::Window | fpl::InitFlags::Audio);
return 0;
}

Initialization with settings

To initialize FPL with custom settings you need to call fpl::InitPlatform() and provide the desired fpl::InitFlags and the fpl::Settings parameter:

fpl::Settings settings = DefaultSettings();
// change the settings here
fpl::InitPlatform(fpl::InitFlags::All, settings);

Release

When you are done, you release FPL with the call fpl::ReleasePlatform():

fpl::ReleasePlatform();

Result/Error checking

There is no guarantee that fpl::InitPlatform() will always work with the fpl::Settings you specified, maybe the audio device does not support a sample rate of 1337 Khz or your video card does not support OpenGL version 3.7 - who knows.

Therefore you should always check the result of the fpl::InitPlatform() call! The result is a simple bool indicating whether the initialization failed or was successful - its that easy.

Also you should release the platform when the initialization was successful only! If something goes wrong the resources are already cleaned up by FPL.

In addition you should use fpl::GetPlatformError() to print out the actual error when the initialization fails!

Bad: (But will work)

if (fpl::InitPlatform(fpl::InitFlags::All)) {
// your code here
}
fpl::ReleasePlatform();

Good:

if (fpl::InitPlatform(fpl::InitFlags::All)) {
// your code here
fpl::ReleasePlatform();
}

Better:

if (fpl::InitPlatform(fpl::InitFlags::All)) {
// your code here
fpl::ReleasePlatform();
} else {
const char *errStr = fpl::GetPlatformError();
fpl::console::ConsoleFormatError("FPL-ERROR: %s\n", errStr);
}

See the Error handling page for more details about error handling.

Tips

After releasing FPL you can call fpl::InitPlatform() again if needed - for example: Finding the proper audio device, Testing for OpenGL compability, etc. may require you call InitPlatform() and ReleasePlatform() multiple times.
